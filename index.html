      Metadata dashboard

      Your CSV is summarized into descriptive metadata (statistics and structure — not a copy of every cell). The export includes full-table marginals for synthesis (numeric histograms and category proportions from every row in the working table). Use Edit metadata on any column to adjust labels, how the field should be treated, or overrides for the synthesis step.

      1. Parse — Read headers and cell values from your session.
      2. Profile — Counts, types, and distributions per column.
      3. Adjust — Include / exclude and edits shape the JSON contract.
      4. Generate — The next step will consume the JSON (see dropdown for the full export).

      Synthetic targets have been customized (columns and/or correlations).

        Columns

        Each card is one field in the metadata package. Toggle Include to drop it from synthesis metadata only; your uploaded table in this browser session is unchanged.

        Numeric correlations

        Pairwise Pearson correlations use rows where both values parse as numbers (same row alignment). Large files scan the first segment of rows for speed; very wide numeric tables show a subset of columns. The matrix shows synthetic targets used for generation (cell colors follow each value). Click an off-diagonal cell to edit a target; observed r from your file appears in the cell tooltip and in the list below when you override a pair.

        Distributions

        Adjust how synthetic data should follow marginal shapes: pick a numeric distribution shape, drag histogram bins, and set category strategy (balance, merge rare, or custom sliders). Amber shows the observed baseline from your file; blue is your synthetic target. When this section differs from defaults, Revert changes appears at the top right of the accordion.

        Technical JSON metadata (full export)

        Open only when you need the exact payload for APIs or documentation. This object is what downstream tools should consume — it contains no raw row dump.
