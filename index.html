    After upload, the inspection step sends column statistics, deterministic checks, and a small row sample to OpenAI through this app’s backend for a written assessment.

      Metadata dashboard

      Your CSV is summarized into descriptive metadata (statistics and structure — not a copy of every cell). The cards below are what a synthetic generator would use to match shape and marginals. Use Edit metadata on any column to adjust labels, how the field should be treated, or notes for the synthesis step.

      1. Parse — Read headers and cell values from your session.
      2. Profile — Counts, types, and distributions per column.
      3. Adjust — Include / exclude and edits shape the JSON contract.
      4. Generate — The next step will consume the JSON (see dropdown for the full export).

      Synthetic targets have been customized (columns and/or correlations).

        Columns

        Each card is one field in the metadata package. Toggle Include to drop it from synthesis metadata only; your uploaded table in this browser session is unchanged.

        Numeric correlations

        Pairwise Pearson correlations use rows where both values parse as numbers (same row alignment). Large files scan the first segment of rows for speed; very wide numeric tables show a subset of columns. Observed is from your file; Synthetic targets is what generation uses — click an off-diagonal cell in the right table to change a target (left table stays read-only).

        Distributions

        Charts reflect the same aggregates that drive histogram- and frequency-based synthetic sampling.

        Technical JSON metadata (full export)

        Open only when you need the exact payload for APIs or documentation. This object is what downstream tools should consume — it contains no raw row dump.
